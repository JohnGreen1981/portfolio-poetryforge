# PoetryForge

## Назначение

Репозиторий CLI-инструмента для анализа русской силлаботонической поэзии.

Проект показывает внешний валидатор для LLM: модель генерирует текст, `poetryforge` проверяет метр, ударения и рифму, затем модель или человек исправляет проблемные строки.

## Стек

- Python 3.12+
- uv
- click
- ruaccent
- pymorphy3
- pytest

## Структура

```text
poetryforge/       исходный код пакета
  cli.py           CLI-команды
  phonetics/       слоги, ударения, метр, фонетика
  rhyme/           проверка и подбор рифм
  data/            маленький пользовательский словарь; большие индексы не входят в git
tests/             тесты
pyproject.toml     пакет и зависимости
uv.lock            lockfile
```

## Роль проекта

Проект показывает инженерный кейс с тестами и понятной задачей проверки LLM-результата: валидации, метриках качества, CLI-упаковке и цикле "LLM -> verifier -> исправление".

## Данные

Большие файлы `words.txt`, `words_expanded.txt`, `rhyme_index.json` и `deep_rhyme_index.json` не коммитятся в clean repo. Они считаются генерируемыми данными или release artifacts.

`poetryforge/data/user_dict.json` должен оставаться нейтральным и не содержать личных рабочих словарей.

## Правила

- Не коммитить `.venv`, кэши, локальные агентные файлы, черновики стихов и приватные корпуса текстов.
- Не добавлять большие generated data в git без отдельного решения.
- Перед публикацией запускать `uv run pytest`.
- Проверять изменения на отсутствие секретов.
- `AGENTS.md` и `CLAUDE.md` должны оставаться синхронизированными по смыслу.
